# Runbook / Roadmap — "where to next?" (ทริปหน้าไปไหน)

อัปเดต: 18 ก.ค. 2026 · เจ้าของ: Nim (@nimjourney)

## ลำดับงาน (สำคัญสุดก่อน)

### 1. แผนของฉัน — ทำให้ใช้จริงได้ก่อน ⭐ งานแรก
เหตุผล: ทริปคาซัคสถานเริ่ม **24 ก.ค.** (อาทิตย์หน้า) ต้องใช้บันทึกจริงระหว่างเที่ยว
- บันทึกรายวันระหว่างทริปให้ลื่น: เพิ่ม/แก้กิจกรรม, จดงบจริงที่จ่าย, แนบรูปต่อวัน
- ข้อมูลต้องไม่หาย (localStorage → ต่อไปมี backend ค่อย sync)
- เอกสาร/ตั๋วเรียกดูง่ายตอนอยู่หน้างาน (ออฟไลน์ได้ยิ่งดี)
- ผลพลอยได้: เนื้อหาจากทริปนี้ → ไปลงแท็บเรื่องเล่าเป็นเรื่องแรก

### 2. เรื่องเล่า — หลังกลับจากทริป
- ✅ เปลี่ยนเป็นระบบ "หนังสือหลายตอน (ตอน/chapter)" — **บันทึกเรื่องวันนี้ (แต่ละวัน) = 1 ตอน** เข้าตรงหน้าเขียนตอน (rich text: B/I/U/S, หัวข้อ, quote, จัดหน้า + รูปไม่เกิน 5 + ลิงก์คลิป) → **ทุกตอนไปรวมที่ สรุปทริป → แท็บ "หนังสือ"** (ปก/สารบัญ/สถานะ + ปุ่มจัดการเล่ม=storyEdit) → หน้าอ่าน chapterRead สะอาดแบบหนังสือ
- ✅ ปุ่ม "สร้างตอนจากบันทึกรายวัน" (ในหน้าจัดการเล่ม) — ดึงเช็คอินแต่ละวันมาเป็นตอน (รูป≤5)
- ต่อยอด: ส่งออกเป็น **ebook / สั่งพิมพ์หนังสือ** (ปุ่มในการ์ดสีเข้มหน้า storyEdit — ยังเป็น "เร็วๆ นี้")
- การจัดหมวดหมู่เรื่องเล่า: **ยังไม่สรุป — คุยกันอีกที**
- ระบบคอมเมนต์ใต้เรื่องเล่า

### 3. วางแผนทริป (ต่อยอดแผนของฉัน)
- สร้างทริปใหม่ตั้งแต่ศูนย์ / ก็อปทริปจริงของคนอื่นมาปรับ
- AI ช่วยจัดวัน (BYOK หรือ AI ของแอปแบบฟรีจำกัด/พรีเมียม)
- ลิงก์จองแอฟฟิลิเอตในหน้าเพิ่มกิจกรรม (ที่พัก/เดินทาง/กิน/ที่เที่ยว)

### 4. สำรวจ — ไว้หลังสุด
Mock ใน UI แล้ว (การ์ดจุดหมาย + 4 หมวด + ปุ่มเพิ่มลงแผน + ป้ายที่มา + ผู้ใช้เพิ่มสถานที่)
แผนข้อมูลจริง:
- แหล่งภายนอก: เริ่มจากสาย OpenStreetMap (Geoapify — ฟรี/ถูก) หรือ Foursquare free tier; เลี่ยง Google Places (~$275/เดือน ไม่มี free tier แล้ว)
- แปลไทย: AI แปล+เรียบเรียงครั้งเดียวต่อสถานที่ → เก็บฐานข้อมูลเราเอง (cache, ไม่แปลสด)
- ผู้ใช้เติมเอง (UGC): ป้าย "จากผู้ใช้จริง" + ตรวจก่อนแสดง + ดึงจากทริป/เรื่องเล่าสาธารณะ
- ต้องมี backend ก่อน (Supabase/Firebase ฟรี tier) — เก็บฐานสถานที่+คำแปล+UGC, ซ่อน API key
- ดึงข้อมูลเป็นล็อตเฉพาะเมืองที่ผู้ใช้ค้นจริง → ค่าใช้จ่ายช่วงเริ่ม ≈ 0–300 บาท/เดือน

## สถานะปัจจุบันของแต่ละแท็บ (mock ใน index.html)
- **หน้าแรก**: บังคับสมัคร/เข้าระบบ (Google / Facebook / อีเมล) จำใน wtn-auth
- **แผนของฉัน**: ทริปคาซัคสถาน 11 วัน (24 ก.ค.–3 ส.ค.) ครบ: รายวัน งบ แผนที่ AI เอกสาร/ตั๋ว
- **เรื่องเล่า**: ฟีดเดิม (ตัวอย่างยามากาตะ) — คงไว้ รอจัดหมวดหมู่
- **สำรวจ**: mock เสร็จตามข้อ 4
- **ฉัน (โปรไฟล์)**: รูปปก/โปรไฟล์ (photo-pick), ตั้งค่าบัญชีและความปลอดภัย (รหัสผ่าน/ผูกบัญชี/ลบบัญชี/ออกจากระบบ), เชื่อม AI key

## ระบบที่มีแล้ว (อย่าทำซ้ำ)
- `photo-pick.js`: เลือกรูปจากเครื่อง + ลาก/ซูม + เปลี่ยน/ลบ — เก็บ `photopick:<id>` (nx,ny,s แบบสัดส่วน) ใน localStorage, ยิง event `photopick-change`, มีโหมด `readonly`+`fallback` ให้การ์ดโชว์เฟรมเดียวกัน
- เช็คอินรายวัน (ปุ่ม "เช็คอิน" ในหน้าทริป): สถานที่+โน้ต+รูปหลายใบจากมือถือ (ย่อรูปเก็บใน `wtn-checkins`) → วัตถุดิบทำรีแคป/เรื่องเล่า
- localStorage keys: `wtn-auth`, `wtn-links`, `wtn-pass`, `wtn-profile` (ชื่อ/ไบโอ/ลิงก์), `wtn-user-pois`, `wtn-checkins`, `wtn-expenses`, `wtn-budget-plan`, `wtn-stories`, `wtn-recap-links`, `wtn-ai-key`, `photopick:*`
- Deploy: แก้ `Where to Next.dc.html` → ก็อปเป็น `index.html` → push GitHub Pages (อย่าลืม `photo-pick.js`)
